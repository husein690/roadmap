الحالات الـ6 للبورت
لما Nmap يسكن بورت، بيرجعلك واحدة من هاي الحالات الـ6. كل حالة إلها معنى مختلف بناءً على الرد اللي استقبله من الهوست. اضغط على أي كارت لتشوف تفاصيل أكثر.
open
مفتوح
البورت شغال وفيه سيرفيس يستقبل connections. هاد أهم هدف لأي pentest.
شو يعني بالنسبة لك:
يعني في تطبيق أو سيرفيس شغال على هاد البورت ويستقبل اتصالات. ممكن تكون web server, SSH, database, أو أي سيرفيس ثاني.

ليش بيظهر open:
• TCP: الهوست أرسل SYN-ACK رداً على SYN بتاعنا
• UDP: الهوست أرسل UDP response
• SCTP: في INIT-ACK response

مثال عملي:
22/tcp open ssh
اضغط للمزيد ▼
closed
مغلق
البورت متاح لكن ما في سيرفيس شغال عليه. الهوست رد بـ RST.
شو يعني:
البورت وصله الباكيت بس ما في خدمة شغالة. مفيد جداً لأنه يأكدلك إن الهوست حي وشغال.

ليش بيظهر closed:
الهوست رد بـ TCP RST flag (Reset). هاد يعني "ما في شي هون".

الفايدة العملية:
حتى لو مغلق، يفيدك تعرف الهوست شغال. بعض الأوقات بتقدر تستغل هاد لـ OS fingerprinting.

21/tcp closed ftp
اضغط للمزيد ▼
filtered
محجوب بجدار ناري
Nmap ما قدر يحدد الحالة. في فايروول بيبلوك الباكيتات ويمنع الرد.
ليش صعب:
لأنه ما في رد خالص (drop) أو في رد error مبهم، فـ Nmap ما يعرف إذا البورت مفتوح أو مغلق.

نوعين من الـ filtered:
• Dropped: الفايروول يحذف الباكيت بدون ما يرد — Nmap بيستنى طويل وبيبعث أكثر من مرة
• Rejected: الفايروول يرد بـ ICMP error (مثل type 3) — أسرع لكن كشف

علامة عملية:
لما السكان ياخد وقت طويل (مثل 2 ثواني بدل 0.05 ثانية) = filtered

139/tcp filtered netbios-ssn
اضغط للمزيد ▼
unfiltered
غير محجوب (لكن غير محدد)
البورت متاح ومش محجوب، لكن Nmap ما يقدر يحدد إذا مفتوح أو مغلق.
متى يظهر:
فقط في ACK scan (-sA). هاد النوع من السكان مصمم لـ firewall rule mapping مش لاكتشاف البورتات المفتوحة.

الفايدة:
يساعدك تفهم قواعد الفايروول — أي بورتات بيعدّيها وأي ما بيعدّيها.

الفرق عن open:
open = عارف إنه شغال
unfiltered = عارف إنه وصله الباكيت بس مش عارف إيش الحالة بالضبط

النوع يظهر مع: nmap -sA

مفتوح أو محجوب — مش واضح
Nmap ما يقدر يميز: إما مفتوح وما رد، أو في فايروول يبلوك.
متى يظهر:
غالباً في UDP scan وبعض TCP scans. لما ما في رد خالص — Nmap ما يعرف إذا البورت مفتوح وما رد على الـ probe، أو في فايروول قاطع.

مثال UDP:

كيف تفرق:
استخدم -sV (version scan) — بيحاول يتفاعل مع السيرفيس ويأكدلك إذا مفتوح فعلاً.

متى يظهر:
فقط في IP ID idle scan (-sI). هاد نوع scan متقدم جداً بيستخدم جهاز وسيط "zombie host" لإخفاء هوية المهاجم.

الـ IP ID idle scan:
تقنية stealth متقدمة — بتبعث باكيتات وكأنها جاية من جهاز ثاني (الزومبي) وبترصد تغيير في IP ID counter ع الزومبي لتعرف الحالة.

الاستخدام العملي:
نادر في pentesting اليومي. أكثر شي بتشوفه في CTFs أو advanced stealth operations.

اكتشاف بورتات TCP
Nmap عنده طريقتان رئيسيتان لسكان TCP. الفرق بينهم مهم جداً للـ stealth وللدقة.

🔪 SYN Scan (نص اتصال)

nmap -sS [target]
  • الافتراضي لما تشغّل Nmap كـ root
  • يبعث SYN فقط — ما يكمل الـ handshake
  • لما يستقبل SYN-ACK يبعث RST ليقطع
  • أسرع وأخفى — ما بيسجله كثير من الـ IDS
  • يحتاج صلاحية root/sudo
⚡ يُسمى Half-Open Scan
لأنه ما بيكمل الـ Three-Way Handshake. بس ما يعني إنك غير مكشوف!

🤝 Connect Scan (اتصال كامل)

nmap -sT [target]
  • يكمل الـ Three-Way Handshake بالكامل
  • SYN → SYN-ACK → ACK (اتصال كامل)
  • أبطأ وأوضح — بيسجّل في logs الهوست
  • يشتغل بدون root
  • أدق — لأنه اتصال حقيقي
🎯 متى تستخدمه؟
لما ما عندك root، أو تحتاج دقة عالية، أو تعمل scan من وراء proxy.
🤝 الـ Three-Way Handshake — كيف تشتغل الـ TCP
SYN Scan (-sS)
1️⃣ أنت → SYN
2️⃣ الهوست → SYN-ACK
3️⃣ أنت → RST (قطع) ❌
✓ ما اكتمل الاتصال = أخفى
Connect Scan (-sT)
1️⃣ أنت → SYN
2️⃣ الهوست → SYN-ACK
3️⃣ أنت → ACK ✅
✓ اتصال كامل = أدق
🔍 مثال حقيقي من --packet-trace على بورت 21 (closed)
sudo nmap 10.129.2.28 -p 21 --packet-trace -Pn -n --disable-arp-ping # ما أرسلناه: SENT (0.0429s) TCP 10.10.14.2:63090 > 10.129.2.28:21 S ttl=56 seq=1699105818 ^أنت ^sourcePort ^الهدف ^port ^SYN flag # ما استقبلناه: RCVD (0.0573s) TCP 10.129.2.28:21 > 10.10.14.2:63090 RA ttl=64 win=0 ^الهدف ^port ^أنت ^RST+ACK = مغلق! PORT STATE SERVICE 21/tcp closed ftp
🔑 فهم الـ flags
S = SYN | A = ACK | R = RST | F = FIN | P = PUSH | U = URG
RA = RST + ACK معاً = الهوست رفض الاتصال لأن البورت مغلق

اكتشاف بورتات UDP
UDP بروتوكول مختلف تماماً عن TCP — بدون connection، بدون handshake، وصعب جداً تسكنه. كثير من المسؤولين بينسوا يحموا UDP!
⚠️ UDP بطيء جداً
سكان أعلى 100 UDP بورت ممكن ياخد 98 ثانية مقارنةً بثانية واحدة لـ TCP. السبب: ما في response لتأكيد الاستقبال، فـ Nmap بيستنى timeout لكل بورت.

TCP — Connection-oriented

  • Handshake قبل البيانات
  • ضمان وصول البيانات
  • رد فوري (SYN-ACK أو RST)
  • سهل السكان ✓
  • SSH, HTTP, FTP, SMB

UDP — Connectionless

  • ما في handshake
  • ما في ضمان للوصول
  • ممكن ما في رد نهائياً
  • صعب السكان ✗
  • DNS, DHCP, SNMP, NetBIOS

💡 نصيحة pentest
UDP ports مهملة كثير في الحماية. DNS (53), SNMP (161/162), TFTP (69), DHCP (67/68) — هاي ممرات شهيرة للاستغلال لأن كثير من المسؤولين بيركزوا على TCP بس!
TCP Flags — أعلام TCP
كل باكيت TCP يحمل flags في header بتحدد الغرض من الباكيت. فهمها ضروري لفهم كيف يشتغل Nmap وكيف تقرأ الـ packet traces.
الـ Flagالاسم الكاملالاستخدامفي Nmap
SYN (S) Synchronize بدء اتصال TCP جديد -sS يبعث SYN فقط
ACK (A) Acknowledge تأكيد استقبال باكيت -sA يبعث ACK فقط
RST (R) Reset إنهاء اتصال فجأة / رفض رد الهوست على بورت مغلق
FIN (F) Finish إنهاء اتصال بشكل نظيف -sF يبعث FIN
PSH (P) Push إرسال البيانات فوراً نادر في scanning
URG (U) Urgent بيانات عاجلة تُعالَج أولاً نادر
SA SYN+ACK رد الهوست على SYN يعني البورت open
RA RST+ACK رفض الاتصال يعني البورت closed

Cheatsheet — كل الأوامر المهمة
ملخص كل الأوامر في الدرس + أوامر مهمة ستحتاجها في الـ pentesting.
🎯 اكتشاف البورتات
-p- سكان كل 65535 بورت -p 22,80,443 سكان بورتات محددة -p 1-1024 سكان نطاق -F أسرع 100 بورت --top-ports=100 أكثر 100 بورت استخداماً
🔍 أنواع السكان
-sS SYN scan (نص اتصال) — الافتراضي مع root — أخفى -sT Connect scan (اتصال كامل) — بدون root — أدق -sU UDP scan — بطيء لكن يكشف سيرفيسز مخفية -sA ACK scan — لرسم خريطة قواعد الفايروول -sV Version scan — يكتشف إصدار السيرفيس
🔧 خيارات مهمة
-Pn لا تعمل ping — تعامل مع الهوست كأنه شغال -n لا تحل DNS — أسرع وأخفى --disable-arp-ping لا تستخدم ARP --packet-trace شوف كل باكيت بيتبعث ويتستقبل --reason شوف سبب الحالة --max-retries=1 قلل retries للسرعة
💾 حفظ النتائج
-oN scan.txt حفظ نص عادي -oX scan.xml حفظ XML (للأدوات) -oG scan.gnmap حفظ grepable -oA scan حفظ كل الأنواع معاً
⚡ أوامر جاهزة للـ Pentest
# اكتشاف سريع أول ما تدخل sudo nmap -sS -Pn -n --disable-arp-ping -p- --min-rate 5000 -oA full_scan TARGET # سكان دقيق مع version detection sudo nmap -sV -sC -Pn -n -p 22,80,443,445 --packet-trace TARGET # UDP للسيرفيسز المهمة sudo nmap -sU -Pn -n -p 53,161,162,69,67 --reason TARGET # فحص الفايروول sudo nmap -sA -Pn -n -p- TARGET
🚨 تحذير مهم
لا تستخدم هالأوامر إلا على أجهزة عندك إذن صريح بتستها! في الـ HTB والـ CTFs يكون عندك إذن. في الواقع بدونه = جريمة في أغلب الدول.

ليش Performance مهمة؟
لما تعمل nmap scan، في ترادوف دايم بين السرعة والدقة والـ stealth. كل مشروع pentest إله احتياجاته: أحياناً بدك تنتهي بسرعة، أحياناً ما تكشف حالك، وأحياناً الشبكة بطيئة وبدك Nmap يتكيف.
⚠️ الـ Default مش دايماً الأفضل
الإعدادات الافتراضية لـ Nmap معقولة، لكن في بيئات الـ pentesting الحقيقية أو الـ CTF محتاج تعدّل حسب الموقف. سكان كل البورتات (-p-) بالإعدادات الافتراضية ممكن ياخد ساعة!

القالب الاسم متى تستخدمه التأثير على الوقت
-T0 / paranoid بارانويد تتهرب من IDS متقدم — ينتظر 5 دقائق بين كل بورت ساعات أو أيام
-T1 / sneaky متسلل تتهرب من IDS — ينتظر 15 ثانية بين كل بورت ساعات
-T2 / polite مؤدب ما تحمّل الشبكة — 0.4 ثانية بين كل بورت دقائق طويلة
-T3 / normal عادي ★ افتراضي الاستخدام العادي — توازن بين السرعة والدقة متوازن
-T4 / aggressive عدواني شبكات سريعة موثوقة — الأشهر في CTFs والـ labs سريع
-T5 / insane مجنون شبكة محلية فقط — ممكن يفوّت بورتات بسبب السرعة سريع جداً
💡 الأكثر استخداماً في الـ CTFs
-T4 هو الأشهر — سريع كفاية ودقيق كفاية على شبكات الـ labs. ما تستخدم T5 إلا لو الشبكة محلية عندك مباشرة.
⚠️ -T0 و -T1 في الواقع
نادراً ما تحتاجهم في الـ CTF. في الـ real world pentest ممكن تستخدمهم لو الـ IDS حساس جداً — لكن بعضهم الأحدث يكتشف أنماط السكان البطيئة أيضاً.
RTT Timeout — وقت انتظار الرد
RTT = Round Trip Time = الوقت اللي ياخده الباكيت ليروح للهدف ويرجع. Nmap يحسب هاد الوقت تلقائياً ويعدّل عليه. لكن تقدر تتحكم فيه يدوياً لتسرّع أو تدقّق السكان.
📡 كيف يشتغل الـ RTT؟
Nmap يقيس زمن الرد من أول باكيتات يبعثها، ثم يعدّل الـ timeout تلقائياً. لو الشبكة بطيئة، بيزيد الـ timeout. لو سريعة، بيقلله.
مثال: كيف Nmap يحسب RTT
SENT (0.0429s) TCP → 10.129.2.28:21 RCVD (0.0573s) TCP ← 10.129.2.28:21 RTT = 0.0573 - 0.0429 = 0.0144s = ~14ms Nmap يستخدم هاد الرقم كـ baseline لحساب الـ timeout

الافتراضي — طويل جداً على شبكة سريعة
الخيارالقيمة الافتراضيةالاستخدام
--initial-rtt-timeout مبني على الشبكة حدد نقطة بداية Nmap لحساب الـ RTT. مثال: --initial-rtt-timeout 50ms
--max-rtt-timeout 10000ms الحد الأقصى للانتظار. قلّله على شبكات سريعة لتسريع السكان. مثال: --max-rtt-timeout 100ms
--min-rtt-timeout 100ms الحد الأدنى — Nmap لن يقل عنه حتى لو الشبكة سريعة
⚠️ خطر تقليل RTT كثيراً
لو حطيت --max-rtt-timeout صغير جداً على شبكة فيها latency، Nmap راح يعتبر كثير من البورتات المفتوحة كـ "timeout" ويظهرها غلط. النتيجة: false negatives — بورتات مفتوحة مش ظاهرة!
Rates & Retries — سرعة الإرسال والإعادة
أقوى طريقة للتحكم المباشر بسرعة Nmap: كم باكيت في الثانية يبعث، وكم مرة يعيد المحاولة على البورتات اللي ما ردّت.
🚀 --min-rate
أقل عدد باكيتات في الثانية — Nmap لن يبعث أقل من هاد الرقم.
--min-rate 300 # 300 pkt/s --min-rate 1000 # 1000 pkt/s (سريع) --min-rate 5000 # 5000 pkt/s (للـ labs)
⚡ تأثير مباشر على وقت السكان
سكان 65535 بورت:
300 pkt/s → ~218 ثانية (~3.6 دقيقة)
5000 pkt/s → ~13 ثانية ⚡
🔁 --max-retries
كم مرة Nmap يعيد إرسال الباكيت لبورت ما ردّ عليه.
--max-retries 10 # افتراضي --max-retries 3 # أسرع، أقل دقة --max-retries 0 # سريع جداً بس خطر
⚠️ تقليله يسرّع لكن يزيد false negatives
مع filtered ports:
retries=10, timeout=100ms → 1 ثانية لكل بورت filtered
retries=1, timeout=100ms → 0.2 ثانية لكل بورت filtered

--min-rate <num> تلقائي أقل معدل إرسال (packets/sec). مثال: 300

Firewall and IDS/IPS Evasion
كيف تتخطى الجدران النارية وأنظمة الكشف — تفصيل كامل
Firewall vs IDS vs IPS — شو الفرق؟
قبل ما نتعلم كيف نتخطاهم، لازم نفهم كيف يشتغل كل واحد منهم وشو الفرق بينهم.

🧱 Firewall (جدار ناري)

بيفلتر الحركة بناءً على قواعد محددة مسبقاً — IP، port، protocol. إما يسمح أو يبلوك أو يرفض الباكيت.

أنواعه:
• Packet Filter — يفحص header فقط
• Stateful — يتابع حالة الاتصال
• Application Layer (WAF) — يفهم محتوى البروتوكول

👁️ IDS (Intrusion Detection System)

يراقب الحركة ويكتشف الأنماط المشبوهة ويُنبّه المسؤول. ما يبلوك — بس يجرس الإنذار.

طريقة الكشف:
• Signature-based — يقارن بأنماط معروفة
• Anomaly-based — يكتشف السلوك الغير طبيعي

🛡️ IPS (Intrusion Prevention System)

مثل IDS لكن يبلوك تلقائياً لما يكتشف هجوم — بدون تدخل بشري.

الفرق عن IDS:
IDS = يشوف ويحكيلك
IPS = يشوف ويبلوك بنفسه

🔀 الفرق العملي للـ Pentester

الـ Firewall بيمنعك من الوصول. الـ IDS/IPS بيكشف إنك تسكن ويُنبّه أو يبلوك.

هدفك:
1. تتجاوز الفايروول = تصل للبورتات
2. تخدع IDS/IPS = ما تكشف نفسك

Decoys — الطُعم والتمويه
أقوى تقنية لإرباك الـ IDS — تبعث السكان وكأنه جاي من عدة IPs في نفس الوقت. المسؤول يشوف كثير من IPs تسكن فما يعرف أيهم أنت!
👥 كيف تشتغل الـ Decoys؟
Nmap يبعث باكيتات بـ IPs مزيفة بالإضافة لـ IP الحقيقي. الهدف يستقبل طلبات من كل الـ IPs ويرد عليها — لكن الردود على الـ IPs المزيفة بتروح ع الفاضي.

# Decoys عشوائية — Nmap يختار IPs عشوائية sudo nmap 10.129.2.28 -D RND:5 # RND:5 = استخدم 5 IPs عشوائية كـ decoys # Decoys محددة يدوياً + أنت في المنتصف sudo nmap 10.129.2.28 -D 10.10.10.1,10.10.10.2,ME,10.10.10.3 # ME = IP الحقيقي بتاعك في المنتصف # مع SYN scan + packet trace لترى الفرق sudo nmap 10.129.2.28 -D RND:5 -sS --packet-trace -p 80
📦 مثال packet-trace مع Decoys
SENT TCP 1.2.3.4:65320 > 10.129.2.28:80 S ← decoy 1 SENT TCP 5.6.7.8:65320 > 10.129.2.28:80 S ← decoy 2 SENT TCP 10.10.14.2:65320 > 10.129.2.28:80 S ← أنت (الحقيقي) SENT TCP 9.8.7.6:65320 > 10.129.2.28:80 S ← decoy 3 RCVD TCP 10.129.2.28:80 > 10.10.14.2:65320 SA ← رد عليك فقط (السريع)
⚠️ قيود الـ Decoys
• الـ IPs المزيفة لازم تكون online — لو الـ decoy IP مش شغال، قد يكشفك
• لا تشتغل مع TCP Connect scan (-sT) بالكامل — بس مع SYN
• بعض الـ IDS الحديثة تكتشف نمط الـ decoys
Source Port Manipulation — تزوير منفذ المصدر
كثير من الفايروولات تثق ببعض البورتات المعروفة (53, 80, 443) وتسمح لحركتها بالمرور. لو غيّرت source port بتاعك لواحد منهم، ممكن تعدّي الفلتر!

🔑 المنطق

الفايروول بيشوف: "هاد الباكيت جاي من port 53 (DNS) — يبدو طبيعي" ويسمح له بالمرور حتى لو هو في الحقيقة Nmap scan.

🎯 أكثر البورتات فاعلية

53 — DNS (الأشهر)
80 — HTTP
443 — HTTPS
67/68 — DHCP
# تغيير source port لـ 53 (DNS) sudo nmap 10.129.2.28 -g 53 # أو sudo nmap 10.129.2.28 --source-port 53 # مثال كامل مع SYN scan sudo nmap 10.129.2.28 -sS -g 53 -p 50000 --packet-trace -Pn -n # مثال كامل مع ncat بعد اكتشاف البورت ncat -nv --source-port 53 10.129.2.28 50000 # إذا الفايروول بيسمح DNS يجوز يتصل!
📦 مثال عملي
بورت 50000 كان filtered مع سكان عادي، لكن مع source port 53:
# بدون source port manipulation: PORT STATE SERVICE 50000/tcp filtered ibm-db2 # مع --source-port 53: SENT TCP 10.10.14.2:53 > 10.129.2.28:50000 S RCVD TCP 10.129.2.28:50000 > 10.10.14.2:53 SA PORT STATE SERVICE 50000/tcp open ibm-db2 # البورت مفتوح فعلاً! الفايروول كان يبلوك فقط الـ source ports العادية
💡 نصيحة مهمة جداً
بعد اكتشاف source port يشتغل، استخدم ncat أو nc بنفس الـ source port للاتصال الفعلي بالسيرفيس — مش بس للسكان!

-ff 16 bytes تجزئة أكثر

Scan Types للتخفي
بعض أنواع السكان تعتمد على "استغلال ثغرات" في تطبيق TCP للكشف عن البورتات بطريقة غير مباشرة — يصعب على الـ IDS تصنيفها كـ port scan.
-sA (ACK)
يبعث ACK فقط — ما يحدد إذا مفتوح أو مغلق، لكن يكشف قواعد الفايروول. لو رد RST = unfiltered. لو ما رد = filtered.
خفي ✓
-sW (Window)
مثل ACK scan لكن يفحص TCP Window size في الرد. على بعض الأنظمة يفرق بين open وclosed من خلال الـ window value.
خفي ✓
-sN (NULL)
يبعث TCP packet بدون أي flag. RFC يقول: لو مغلق يرد RST، لو مفتوح أو filtered يصمت. يخدع بعض الفايروولات البسيطة.
خفي ✓
-sF (FIN)
يبعث FIN flag فقط — كأنك تحاول تغلق اتصال ما بدأ. نفس منطق NULL scan. يتجاوز فلاتر SYN.
خفي ✓
-sX (Xmas)
يبعث FIN+PSH+URG — "كل الأعلام ماشية" كأنها شجرة عيد ميلاد. نفس التأثير. الاسم مصدره أن الـ flags "تضيء" كلها.
خفي ✓
-sM (Maimon)
يبعث FIN+ACK. سُمي على اسم مكتشفه Uriel Maimon. على BSD systems قديمة يميز المفتوح من المغلق.
نادر
-sI (Idle)
IP ID Idle scan — يستخدم جهاز "zombie" وسيط. أنت ما تبعث للهدف مباشرة. الأخفى على الإطلاق لكن الأصعب إعداداً.
أخفى ⭐
💡 NULL/FIN/Xmas — كيف يعملوا
بورت مغلق:
أنت → FIN/NULL/Xmas
الهوست ← RST
= closed
بورت مفتوح أو filtered:
أنت → FIN/NULL/Xmas
الهوست: صمت (لا رد)
= open|filtered
⚠️ لا تشتغل على Windows!
Windows تخرق RFC TCP وترد بـ RST على كل الـ flags الغريبة — حتى للبورتات المفتوحة. هاي السكانات مفيدة فقط على Linux/Unix.
DNS & Proxy Tricks
تقنيات إضافية لإخفاء هويتك أو تجاوز قيود الشبكة.
1️⃣ DNS Server Manipulation

بشكل افتراضي، Nmap يحل أسماء الـ DNS لكل IP يكتشفه — وهاد الطلب DNS يظهر في logs. تقدر تعطيه DNS server خاص بدل الـ default.

# استخدم DNS server محدد sudo nmap 10.129.2.28 --dns-server 8.8.8.8 # مفيد لو الـ internal DNS server يلوغ طلبات الحل # وقف DNS resolution كلياً (أسرع وأخفى) sudo nmap 10.129.2.28 -n # -n = لا تحل أي domain names
2️⃣ Source IP Spoofing

تغيير الـ IP المصدر كلياً. مفيد لو تريد تظهر وكأنك جهاز آخر على الشبكة الداخلية.

# تغيير source IP sudo nmap 10.129.2.28 -S 10.10.10.200 # ⚠️ ما راح تستقبل الردود! الردود بتروح على 10.10.10.200 # مفيد فقط للتمويه — مش للسكان الفعلي # تحديد الـ network interface sudo nmap 10.129.2.28 -e eth0
3️⃣ Proxychains

توجيه السكان عبر proxy أو TOR. أقوى تقنية للإخفاء الحقيقي — لكن بطيئة جداً.

# نمرر nmap عبر proxychains proxychains nmap -sT -Pn -n 10.129.2.28 -p 80,443 # ⚠️ يشتغل فقط مع TCP Connect scan (-sT) # SYN scan (-sS) لا يشتغل عبر proxychains
🎯 ملخص: متى تستخدم إيش؟
فايروول بسيط بيبلوك SYN: جرب -sA أو -sF أو -sN
فايروول يبلوك حسب source port: جرب -g 53 أو --source-port 443
IDS يكشف السكانات: جرب -D RND:5 + -T1
تريد تخفي IP بالكامل: proxychains أو -D مع IPs حقيقية
IDS يعرف signature الباكيت: جرب --data-length أو --mtu
