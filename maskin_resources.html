🔷 ما هو RISC-V؟

أي processor يحتاج "لغة" يفهمها — هاي اللغة اسمها Instruction Set Architecture (ISA). تحدد الأوامر المتاحة، الـ Registers، وكيف يتعامل مع الذاكرة.

Intel/ARM (مغلق)
شركات خاصة
لازم تدفع رسوم ترخيص
ما تقدر تعدّل عليه
RISC-V (مفتوح) ✅
مجاني 100%
أي شركة تقدر تبني عليه
قابل للتعديل والتوسيع
💡 مثال: CH32V307 (الـ lab kit) + Raspberry Pi Pico 2 = كلاهم RISC-V → نفس الكود يشتغل على الاثنين!
🔧 Variants والـ Extensions

RISC-V = Base (47 أمر فقط) + Extensions اختيارية حسب الحاجة.

Pseudo Instructions = اختصارات تكتبها أنت، الـ Assembler يترجمها تلقائياً لأوامر حقيقية. ما تحتاج تحفظ كيف تتترجم — فقط تعرف كيف تستخدمها!

💾 Load/Store Architecture

المبدأ الأساسي: الأوامر الحسابية تشتغل على Registers فقط! الذاكرة = فقط عبر Load/Store.

x86 (Intel) — معقد
add r0, M(r1) # جمع مع الذاكرة مباشرة!
RISC-V — بسيط ✅
lw x3, 0(x2) # جيب من الذاكرة add x1, x1, x3 # احسب

► أوامر Store

📐 Sign Extension — ليش مهم؟

في الذاكرة: unsigned char a = 251 و signed char b = -5 — كلاهم يُخزَّن كـ 0xFB! لما تحمّل للـ register (32-bit) لازم تحدد النوع:

# -5 بالـ 8-bit = 1111 1011 = 0xFB # MSB = 1 → سالب (Two's Complement) lbu t0, 0(t0) # zero extend → t0 = 0x000000FB = 251 ❌ خطأ للسالب! lb t1, 0(t1) # sign extend → t1 = 0xFFFFFFFB = -5 ✅ صح!
💡 القاعدة: MSB=1 → سالب → sign extend يحشي بـ 1. MSB=0 → موجب → يحشي بـ 0.
✅ Sign Extension = محافظة على المعنى عند التكبير من 8-bit أو 16-bit لـ 32-bit.
⛓️ Instruction Cycle — كيف ينفذ المعالج الكود
# كل cycle يمر بـ 4 مراحل: 1. FETCH ← جيب الأمر من عنوان PC ↓ PC += 4 (كل أمر = 4 bytes) 2. DECODE ← فهّم الأمر (add؟ load؟ branch؟) ↓ 3. EXECUTE ← نفّذ العملية (ALU يحسب) ↓ 4. WRITEBACK← احفظ النتيجة في register أو ذاكرة
💡 بالنظرية = 4 cycles/أمر. بالواقع = ~1 cycle/أمر بسبب Pipelining (خارج نطاق الكورس).
🔢 تحميل أرقام كبيرة — LUI + ADDI

addi عندها 12-bit فقط للـ immediate → أكبر قيمة = 2047. لو الرقم أكبر، الـ assembler يقسمه لأمرين:

# li t0, 10 → يكفي 12-bit: addi t0, zero, 10 # li t0, 1000000 → ما يكفي! يتحول لـ: lui t0, 0xf4 # حط 0xf4 في الـ 20 bits العليا → t0 = 0x000F4000 addi t0, t0, 0x240 # أضف الـ 12 bits السفلى → t0 = 0x000F4240 = 1,000,000
📝 Variables في Assembly
# بدل العناوين المطلقة، استخدم labels: la x1, var_a # x1 = عنوان var_a lb x2, 0(x1) # x2 = قيمة var_a = 10 la x1, var_b lb x3, 0(x1) # x3 = قيمة var_b = 20 add x1, x2, x3 # x1 = 30 var_a: .byte 10 var_b: .byte 20
⚠️ Memory Alignment — القاعدة الذهبية
الأمرالحجمشرط العنوانمثال صحمثال خطأ
lw / sw4 bytesيقسم على 40x1000x102 ❌ كراش!
lh / sh2 bytesيقسم على 20x1020x101 ❌ كراش!
lb / sb1 byteأي عنوان0x1010x103 ✅
# مشكلة شائعة: var_a: .byte 10 # عنوانه 0x100 ✅ var_b: .word 20 # عنوانه 0x101 ❌ كراش! (word لازم يقسم على 4) # الحل: var_a: .byte 10 .align 2 # العنوان الجاي يقسم على 2^2=4 var_b: .word 20 # هلأ عنوانه 0x104 ✅
💥 تكسر قاعدة الـ Alignment = البرنامج يكرش (Exception)!

◈ تمارين المحاضرة 2 — RISC-V Assembly
تمرين 01 — سهل
مجموع من 1 إلى 5
اكتب برنامج يحسب 1+2+3+4+5 ويخزن النتيجة في t0. النتيجة المتوقعة = 15.
استخدم: li, add, addi, blt, label
تمرين 02 — سهل
الضرب بدون mul
احسب 7 × 3 = 21 بدون استخدام أمر mul. استخدم loop + add فقط. تلميح: 7×3 = 7+7+7
counter يبدأ من 0، result يبدأ من 0، كرر إضافة var_a لـ var_b مرات
تمرين 03 — سهل
Variables في الذاكرة
عرّف متغيرين var_a=15 و var_b=8 في الـ .data section. احملهم واحسب الفرق (var_a - var_b) واخزن النتيجة في s0. النتيجة = 7.
استخدم: .data, .word, la, lw, sub
تمرين 04 — متوسط
Sign Extension — lb vs lbu
عرّف byte يساوي 0xFF في الذاكرة. احمله مرتين: مرة بـ lb في t0 ومرة بـ lbu في t1. قارن النتيجتين. ما الفرق ولماذا؟
0xFF = 255 unsigned = -1 signed. lb → 0xFFFFFFFF, lbu → 0x000000FF
تمرين 05 — متوسط
Array في الذاكرة
احفظ الأرقام 10, 20, 30, 40, 50 في الذاكرة باستخدام .word. ثم اقرأهم كلهم باستخدام loop وoffset متزايد، وأضفهم معاً. النتيجة = 150.
استخدم base address + offset (0, 4, 8, 12, 16) — كل word = 4 bytes
تمرين 06 — متوسط
Memory Alignment
اكتب: var_a: .byte 5 ثم var_b: .word 10 مباشرة بعده وشوف إذا يشتغل. بعدين صلح المشكلة باستخدام .align 2 وأثبت إنه شغّال.
.align 2 يعني العنوان الجاي يقسم على 2^2 = 4
تمرين 07 — متوسط
العد التنازلي + حفظ في الذاكرة
اكتب loop يبدأ من 10 وينقص 1 كل مرة حتى يصل للصفر. في كل iteration خزّن القيمة الحالية في array. النتيجة: array تحتوي [10, 9, 8, ..., 1].
احتاج base address + counter يزيد بـ 4 كل iteration
تمرين 08 — صعب
إيجاد الـ Maximum
عندك array من 5 أرقام: [3, 7, 2, 9, 1]. اكتب loop يجد أكبر رقم ويخزنه في s0. النتيجة = 9.
استخدم blt للمقارنة — لو العنصر الحالي أكبر من الـ max الحالي، حدّث الـ max
تمرين 09 — صعب
Byte vs Halfword vs Word — Little Endian
احفظ القيمة 0x12345678 كـ word في الذاكرة. ثم اقرأ: (1) البايت الأول بـ lb، (2) الـ halfword الأول بـ lh، (3) الـ word كاملة بـ lw. ما هي النتائج؟ تذكر Little Endian!
Little Endian: البايت الأصغر في العنوان الأصغر → lb: 0x78, lh: 0x5678, lw: 0x12345678
تمرين 10 — تحدي 🏆
Bubble Sort
اكتب Bubble Sort يرتب array من 5 أرقام تصاعدياً: [5, 3, 8, 1, 4] → [1, 3, 4, 5, 8]. تحتاج nested loops: outer loop + inner loop + swap بين عنصرين.
للـ swap: استخدم register مؤقت. lw عنصرين → sw معكوسين

🦘 Jump Instructions — jal و jalr

أمرين فقط للـ Unconditional Jump في RISC-V. كلمة "Link" تعني احفظ عنوان العودة (PC+4) عشان تقدر ترجع بعدين.

الأمرالمعنىالنتيجة
jal rd, offsetJump and Linkrd = PC+4 ثم PC += offset
jalr rd, offset(rs1)Jump and Link Registerrd = PC+4 ثم PC = rs1+offset
► Pseudo Instructions للـ Jump
Pseudoيتحول لـمتى تستخدمه؟عنوان العودة
j labeljal zero, offsetقفز بسيط بدون عودةيُهمَل (zero)
call labeljal ra, offsetاستدعاء دالة مع عودةمحفوظ في ra
jr rsjalr zero, 0(rs)قفز لعنوان في registerيُهمَل (zero)
retjalr zero, 0(ra)العودة من دالة—
💡 j و call و jr و ret كلهم Pseudo Instructions — الـ Assembler يترجمهم لـ jal أو jalr تلقائياً!
j label — بدون عودة
# j = jal zero, offset # العودة تُكتب في zero = تُهمَل blink: # كود يومض LED j blink # loop أبدي
call label — مع عودة
# call = jal ra, offset # العودة تُحفظ في ra call my_func # يرجع هنا بعد ret
⚠️ jal يقفز +/- 1MB فقط (20-bit offset). لو العنوان أبعد (مثل FLASH memory):
li t0, 0x00000100 | jr t0 ← jalr يقدر يقفز لأي عنوان ✅
🔀 Branching — القفز الشرطي

الـ Branch instructions تقارن مباشرة بدون flag register منفصل — هاد يبسّط الـ Hardware!

► الأوامر الحقيقية (6 أوامر)
الأمرالمعنىالشرط
beq rs1, rs2, labelBranch if Equalrs1 = rs2
bne rs1, rs2, labelBranch if Not Equalrs1 ≠ rs2
blt rs1, rs2, labelBranch if Less Than (signed)rs1 < rs2
bge rs1, rs2, labelBranch if Greater or Equal (signed)rs1 ≥ rs2
bltu rs1, rs2, labelBranch if Less Than (unsigned)rs1 < rs2 (u)
bgeu rs1, rs2, labelBranch if Greater or Equal (unsigned)rs1 ≥ rs2 (u)
► Pseudo Instructions للمقارنة
Pseudoيتحول لـالشرط
bgt rs1, rs2, labelblt rs2, rs1, labelrs1 > rs2 (اعكس الترتيب!)
ble rs1, rs2, labelbge rs2, rs1, labelrs1 ≤ rs2
beqz rs, labelbeq rs, zero, labelrs = 0
bnez rs, labelbne rs, zero, labelrs ≠ 0
bgtz rs, labelblt zero, rs, labelrs > 0
bltz rs, labelblt rs, zero, labelrs < 0
💡 ليش ما في bgt حقيقي؟ لأن (a > b) = (b < a) — اعكس الترتيب وخلص!
⚠️ ما في Immediate versions! ما تكتب bgt t1, 5, end — لازم تحط 5 في register أول: li t2, 5 ثم bgt t1, t2, end
► مثال عملي — Factorial (5! = 120)
li t0, 1 # y = 1 li t1, 1 # i = 1 forloop: li t2, 5 bgt t1, t2, end # إذا i > 5 اخرج (= blt t2, t1, end) mul t0, t0, t1 # y = y × i addi t1, t1, 1 # i++ j forloop end: # t0 = 120 ✅
📚 الـ Stack — مخزن مؤقت

Registers = 32 فقط. لما تنتهي = احفظ مؤقتاً في الـ Stack.
sp (x2) = Stack Pointer، يشير لرأس الـ Stack دايماً.

⬇️ PUSH — احفظ في Stack
# لازم تنزّل sp أول! addi sp, sp, -4 # حجز مكان جديد sw t0, 0(sp) # احفظ t0 # ليش ننزّل أول؟ # sp يشير لآخر قيمة محفوظة # لو حفظت أول = كتبت فوق قيمة!
⬆️ POP — أعد من Stack
# اقرأ أول ثم ارفع sp lw t0, 0(sp) # أعد t0 addi sp, sp, 4 # حرّر المكان # ترتيب POP = عكس PUSH!
► حفظ 3 registers معاً
# PUSH t0, t1, t2: addi sp, sp, -12 # مساحة لـ 3 words (3×4) sw t0, 8(sp) # t0 في الأعلى sw t1, 4(sp) sw t2, 0(sp) # t2 في الأسفل # POP (عكسي): lw t2, 0(sp) lw t1, 4(sp) lw t0, 8(sp) addi sp, sp, 12
💡 Stack ينمو للأسفل: PUSH → sp تنقص، POP → sp تزيد.
🔧 Function Calls — ABI Conventions

Assembly ما عندها functions مدمجة — نبنيها بأنفسنا باتباع ABI Conventions عشان الكود يتوافق مع C والمكتبات.

قواعد ABI للدوال
Parameters: a0-a7 (بالترتيب)
Return Value: a0
Return Address: ra (يحفظه call)
استدعاء: call label
عودة: ret
لو ما حطينا ret؟
البرنامج ما يرجع للمكان الصح
يكمل ينفذ الكود الجاي بالذاكرة
= نتائج عشوائية أو كراش! 💥
► مثال كامل — دالة min(a, b)
# الاستدعاء: li a0, 20 # parameter a = 20 li a1, 10 # parameter b = 10 call min # ra = عنوان العودة، اقفز لـ min # بعد العودة: a0 = 10 ✅ min: blt a0, a1, min_end # إذا a < b ارجع (a0 جاهز) mv a0, a1 # b أصغر → a0 = b min_end: ret # = jalr zero, 0(ra)
💾 Register Saving — من يحفظ ماذا؟

لما تستدعي دالة ما تعرف أي registers تغيّرها! الـ ABI يحل هاي المشكلة بتقسيم الـ registers لنوعين:

🔴 Caller-Saved (مؤقتة)
t0-t6, a0-a7, ra

الدالة المستدعاة تقدر تغيّرها بحرية
لو تحتاجها بعد الـ call = أنت تحفظها

مثل: مكتب مشترك — ما تضمن إن ورقتك تبقى!
🟢 Callee-Saved (محفوظة)
s0-s11, sp

الدالة لو استخدمتها = تحفظها وترجعها
مضمون ما تتغير بعد الـ call ✅

مثل: مكتب خاص — من يستخدمه يرجعه كما كان
⚠️ الضمان مش تقني (hardware) — هو اتفاقية (ABI)! كل دالة مكتوبة صح ملزمة بحفظ s registers وإرجاعها. لو دالة ما تتبع ABI = هي المخطئة.
► المشكلة والحل
# المشكلة: max ممكن تغيّر t2! mv a0, t0 mv a1, t1 call max # t2 ممكن ضاع! ❌ mv a1, t2 # t2 قيمته غير مضمونة! # الحل 1 — احفظ t2 في Stack: addi sp, sp, -4 sw t2, 0(sp) # احفظ t2 call max lw t2, 0(sp) # أعد t2 addi sp, sp, 4 # الحل 2 — استخدم s register (أنظف): mv s0, t2 # انقل لـ s0 (مضمون لا يتغير) call max # s0 مضمون ✅ mv a1, s0 # استخدم القيمة المحفوظة
🔗 Nested Functions — Prologue & Epilogue

لما دالة تستدعي دالة ثانية — call يكتب فوق ra! لازم تحفظه.

# max_of_three(a, b, c) تستدعي max(a, b) مرتين max_of_three: # ═══ PROLOGUE — في البداية: ═══ addi sp, sp, -8 # مساحة لـ s0 و ra sw s0, 4(sp) # احفظ s0 (Callee-Saved) sw ra, 0(sp) # احفظ ra (call سيكتب فوقه!) mv s0, a2 # s0 = c (نحفظه بـ s register) call max # a0 = max(a, b) mv a1, s0 # a1 = c (من s0 المضمون) call max # a0 = max(max(a,b), c) ✅ # ═══ EPILOGUE — قبل الرجوع: ═══ lw s0, 4(sp) # أعد s0 lw ra, 0(sp) # أعد ra addi sp, sp, 8 # أعد الـ stack ret
✅ قاعدة ذهبية: لو دالتك تستدعي دالة ثانية = احفظ ra في الـ Prologue دايماً!
► خطوات كتابة أي دالة
#القاعدةالسبب
1استخدم t registers للحسابات المؤقتةبدون حفظ في Stack
2لو تحتاج قيمة تبقى بعد call = استخدم s registerمضمون ما يتغير
3لو استخدمت s register = احفظه في Prologueأنت callee = ملزم بإرجاعه
4لو دالتك تستدعي دالة ثانية = احفظ racall سيكتب فوقه!
📋 Arrays في Assembly
► تعريف Arrays
# Array من bytes (محددة القيم): numbers: .byte 10, 20, 30, 40, 50 # Array فاضية (5 bytes): numbers: .space 5 # Array من words (5 عناصر = 20 byte): .align 2 numbers: .word 2, 4, 6, 8, 10 # Array words فاضية (5 عناصر): .align 2 numbers: .space 20 # 5 × 4 bytes
💡 .space = عدد الـ bytes دايماً! لو تبي 5 words = .space 20 (5×4)
► الوصول للعناصر
la t0, numbers # t0 = عنوان أول عنصر # Byte array (offset = index): lb t1, 0(t0) # numbers[0] lb t1, 4(t0) # numbers[4] # Word array (offset = index × 4): lw t1, 0(t0) # numbers[0] lw t1, 4(t0) # numbers[1] ← 1×4=4 lw t1, 16(t0) # numbers[4] ← 4×4=16
💡 ليش offset × 4 للـ words؟ لأن الذاكرة = سلسلة bytes. كل word تأخذ 4 bytes متتالية.
► Loop على Array — مجموع 10 أرقام
li t0, 0 # i = 0 li t1, 0 # sum = 0 la t2, numbers # t2 = عنوان أول عنصر loop: slli t3, t0, 2 # t3 = i × 4 (shift left = × 2² = × 4) add t3, t2, t3 # t3 = عنوان numbers[i] lw t3, 0(t3) # t3 = numbers[i] add t1, t1, t3 # sum += numbers[i] addi t0, t0, 1 # i++ li t4, 10 blt t0, t4, loop .align 2 numbers: .word 2,4,6,8,0,1,2,3,4,5
💡 slli t3, t0, 2 = i × 4 بدون mul. Shift Left أسرع لأنه أمر بسيط للـ Hardware!
📤 Arrays كـ Function Parameters

ما ترسل الـ Array كاملة — آلاف bytes في الـ Stack! بدلاً من ذلك أرسل العنوان + الحجم فقط.

main: la a0, numbers # a0 = عنوان أول عنصر li a1, 10 # a1 = حجم الـ array call sum # a0 = مجموع الأرقام # int sum(int array[], int size) sum: li t0, 0 # sum = 0 sum_loop: lw t1, 0(a0) # اقرأ العنصر الحالي add t0, t0, t1 # sum += element addi a0, a0, 4 # انتقل للعنصر الجاي addi a1, a1, -1 # size-- bnez a1, sum_loop # إذا ما خلصنا كرر mv a0, t0 # return sum ret .align 2 numbers: .word 2,4,6,8,0,1,2,3,4,5
